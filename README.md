# KJ Credit Analyst —  기업여신심사 자동화 데모

> (주)H기업 비즈니스파트너론 17.1억 케이스 기반
> 8-Stage 심사프로세스 단계별 처리상황 시각화 데모

## 즉시 실행 (한 줄)

```bash
cd /Users/user/DEV/KJ_Credit_Analysis/demo && python3 -m http.server 8000
```

→ 브라우저에서 **http://localhost:8000** 열기

> ⚠ `index.html` 더블클릭으로 직접 열면 `fetch('data/...json')` 차단으로 로드 실패. 반드시 로컬 웹서버 사용.
> Python 없으면: `npx serve` (Node) 또는 VS Code Live Server 익스텐션 사용.

## 시연 시나리오 (45초)

| 시점 | 동작 |
|---|---|
| 0초 | 우상단 **▶ 심사 시작** 클릭 (또는 스페이스바) |
| 0~5초 | Stage 1 — 좌측 30 파일이 정규화 카탈로그로 변환 (실시간 파일별 처리 상태) |
| 5~12초 | Stage 2-5 **병렬** 진행 (재무·매출채권·타행·담보 4개 카드 동시 진행) |
| 12~20초 | ★ **Stage 6 클라이맥스** — 알람 효과 + 동일차주 그래프 (H기업→두**·에스티**) + "70.5억 검출" |
| 20~25초 | Stage 7 — 위험 가중치 막대 그래프 |
| 25~40초 | Stage 8 — KJ은행 표준 승인신청서가 한 줄씩 자동 채워지는 타이핑 애니메이션 |
| 40~45초 | 하단 메트릭 표시 + HITL 결재 버튼 활성화 |

## 핵심 시연 포인트

### 1️⃣ 좌측 자료 트리 (30 파일)
- 카테고리별 분류 (표준 양식 / 신용평가 / 기초 정보 / 재무제표 / 매출채권 / 타행 / 담보 / 관계회사 / 사업성)
- ★ 표시 파일 = god-node·핵심 단서 (부속명세서·타행거래·관계회사·단기대여원장)
- 처리 진행에 따라 노란색 → 녹색으로 변화

### 2️⃣ 8-Stage 자동 진행
- 각 Stage 카드: 진행률 바 + Agent 모델명 표시
- Stage 2-5는 병렬 (4 카드 동시 진행)
- Stage 6은 빨간 강조 + 알람 흔들림 효과

### 3️⃣ Stage 6 — Killer Feature
- **자동 검출 그래프** (자체 SVG): HST → 00건설(100% 자회사) → ***건설(동일대표) 노드
- 단기대여 합산 70.5억 적색 강조 표시
- 신청 17.1억 단독 평가 대비 누락 위험 명확

### 4️⃣ Stage 8 — KJ은행 표준 양식 자동 작성
- 차주 개요부터 심사 의견까지 8개 섹션이 한 줄씩 타이핑 (커서 깜빡임)
- 모든 항목에 출처 자료 인용 (예: "📄 출처: 5.23년재무제표(감사).pdf p.3")
- 동일차주 합산 검토 섹션 적색 강조
- HITL 버튼 (승인 / 조건부 / 반려) — 최종 결재는 사람이

### 5️⃣ 하단 메트릭
- TTFT 35분 · 표준 양식 일치율 84% · 동일차주 검출 2건 · 근거 인용 28건 · 토큰 비용 $3.2

## 기술 스택

- **HTML5 + Vanilla JS + CSS** — 외부 의존성 0 (CDN·라이브러리 X)
- **SVG 자체 렌더링** — 동일차주 그래프, 차트
- **시뮬레이션** — `setTimeout` 기반 단계별 진행
- **실 LLM 호출 X** — 모든 결과는 `data/hst_dataset.json`에 미리 정의

## 광주은행 가상화 환경 호환성

- 외부 CDN·인터넷 의존성 0 → 망분리 환경에서도 실행 가능
- ZIP으로 압축하여 KJ은행 내부망 PC에 배포 → 가상화 브라우저로 즉시 시연
- `python3` 또는 `node` 가상화 환경에서 사용 가능

## 실 시스템 전환 시 변경 지점

본 데모를 실 시스템으로 전환할 때 수정 대상:

| 변경 지점 | 시뮬레이션 (현재) | 실 시스템 |
|---|---|---|
| `script.js` `runStage()` | setTimeout으로 가짜 진행 | 실제 NCP HCX-007 / SEED-32B API 호출 |
| `data/hst_dataset.json` | 미리 정의된 결과 | LLM 응답 + Vector DB 결과 |
| `renderRelatedPartiesSVG()` | 정적 그래프 | Neo4j Graph DB query → 동적 |
| `typeFormSections()` | 미리 정의 텍스트 타이핑 | 실제 Report Writer Agent 출력 stream |

UI·상태머신·시각 효과는 그대로 재사용 가능. 실 시스템에서도 동일한 UX.
